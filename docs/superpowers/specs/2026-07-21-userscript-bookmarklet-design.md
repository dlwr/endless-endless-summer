# userscript / ブックマークレット版 設計

2026-07-21 ブレインストーミングで確定。

## 背景

Tumblr API のレートリミット緩和申請が却下された。現行の Web アプリ構成(単一 consumer key を全ユーザーで共有、1,000/hr・5,000/day)では公開サービスとしてスケールしない。そこで公開形態を「tumblr.com 上で動くユーザースクリプト/ブックマークレット」に転換する。ユーザー自身のログインセッション(内部 API)を使うため consumer key が不要になり、レートリミットはユーザー単位となって構造的に問題が消える。

2026-07-17 設計時にユーザースクリプト案を却下した理由は「キーボードショートカットの自前実装が必要」だったが、本設計では描画・操作を Tumblr 本体に任せるため、この理由は該当しない。

## 決定事項

| 論点 | 決定 |
|---|---|
| レートリミット回避 | Tumblr セッション相乗り(内部 API `www.tumblr.com/api/v2`) |
| 配布形態 | ユーザースクリプト + ブックマークレットのデュアルビルド(1ソース2形態) |
| 既存 Web アプリ | 自分専用として存続。コアロジックを共有パッケージに切り出す |
| UI | ダッシュボードへの注入(自前 UI は持たない) |
| 注入方式 | API レスポンス介入(fetch フック)。DOM には触らない |
| 置換度合い | 完全置換モード(タイムライン全体をランダム過去ポストにする)。オン/オフトグル付き |

## アーキテクチャ

モノレポに 2 パッケージを追加する。

- `packages/core/`(新規): 年均等ランダムサンプリング、`before` タイムスタンプ計算、最古ポスト日時キャッシュのロジックを Workers 版から切り出す。ストレージ(KV / GM ストレージ / localStorage)と API クライアント(公式 API / 内部 API)はインターフェースで抽象化し、各実装から注入する。既存の該当テストはここへ移動。
- `packages/userscript/`(新規): fetch フック本体、内部 API クライアント、トグル UI、2 形態のビルド。
- 既存 Workers + SPA: `core` を import する形にリファクタし、自分専用として維持。

## 動作フロー

1. ページコンテキストで `window.fetch` をラップする。ユーザースクリプト版は `@run-at document_start` で Tumblr 本体より先にフックする。
2. ダッシュボードタイムライン API(`/api/v2/timeline/dashboard`)へのリクエストを検出したら、そのリクエストのヘッダーから認証情報(Bearer トークン等)を拝借して内部 API を叩く。フォロー一覧 → ランダムブログ選択 → `/api/v2/blog/{name}/posts?before=…` でランダム過去ポストを取得する。取得元も注入先も同じ API v2 形式のため変換はほぼ不要。
3. レスポンスの `timeline.elements` をランダム過去ポストに置換して返す。描画は Tumblr 本体の React が行うため、リブログ・ライク・ノート表示・本家キーボード操作はネイティブに動く。ページング(`_links.next`)は続きが取れるよう自前で維持する。
4. 最古ポスト日時キャッシュは `GM_setValue` があれば使い、なければ `localStorage`(ブックマークレットは自動的に後者)。
5. 1 ページ分に必要な件数しか内部 API を叩かない(ユーザー自身のセッションでも行儀よく)。

## デュアルビルド

esbuild で同一エントリから 2 形態を出力する。

- **`.user.js`**: Tampermonkey ヘッダー付き、`@run-at document_start` で自動起動。初回ロードのタイムラインから置換できる。ページ内トグル + GM メニューでオン/オフ。
- **ブックマークレット**: 全コードを `javascript:(()=>{…})()` に内包した自己完結型。Tumblr の CSP により外部スクリプト読み込みは不可の前提で、一切の外部ロードをしない。クリック起動のため初回表示分は素のまま。起動時に SPA ナビゲーション(pushState でダッシュボードへ再遷移)を蹴ってタイムライン再取得を誘発する。効かなければ「起動後にスクロールで読み込まれる分から置換」にフォールバック。再クリックでオフ。

位置づけ: 常用者にはユーザースクリプト版を推奨し、ブックマークレットは「インストール不要のお試し版」とする。

## エラー処理

フック内で何が起きても必ず素のレスポンスにフォールバックし、ダッシュボードを壊さないことを最優先とする。内部 API の形状変化でパースに失敗したら静かに置換をやめて console に警告を出す。

## テスト

- コアロジック: 既存の Vitest 資産を `packages/core/` へ移して維持。
- レスポンス置換: 実レスポンスをサニタイズした fixture JSON で TDD。
- 実機: Tampermonkey + agent-browser で確認。

## Spike(実装前の実機検証)

以下が成立しなければ設計を再検討する(DOM 注入への後退を含む)。

1. 他ブログのポストを dashboard レスポンスに混ぜたとき、Tumblr の React が正しく描画し、リブログ・ライクが機能するか。
2. 内部 API がセッション認証 + 拝借トークンで叩けるか(フォロー一覧、`/blog/{name}/posts?before=`)。
3. Tumblr の CSP 下で `javascript:` ブックマークレットが実行できるか(ブラウザ仕様上はページ CSP の適用外だが実機確認)。
4. クリック起動後の SPA 再遷移でタイムライン再取得を誘発できるか。

## 配布

GitHub Releases に `.user.js` とブックマークレット(コピー用 HTML ページ等)を置く。README にインストール手順。必要になったら Greasy Fork も検討。
